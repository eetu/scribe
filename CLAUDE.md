# scribe — repo overview

Self-hosted Audible mirror. Sibling product to [halo](../halo) and
[chat](../chat); shares design system but is a separate Rust + React app.

## Layout

```
backend/         axum 0.8 (Pi-side), rusqlite, OIDC kanidm + DEV_AUTH
press/           axum 0.8 (mini-side), ffmpeg subprocess worker
shared/          types shared between app + press (JobSpec, BookMeta, …)
shim/            Python FastAPI sidecar wrapping mkb79/audible — auth + library + voucher
frontend/        Vite + React 19 + Emotion + TanStack Router (file-based)
.claude/skills/  scribe-design skill (visual language, brand)
```

Per-area instructions in `backend/CLAUDE.md`, `press/CLAUDE.md`,
`shim/CLAUDE.md`, `frontend/CLAUDE.md`.

## Conventions

- **Auth model.** Same as chat — kanidm OIDC, `DEV_AUTH=1` fallback for
  local work. Every `/api/*` route except `/api/me` requires session cookie.
  Per-user data isolation via `user_sub` claim — no admin role, no shared
  state.
- **Audible state isolation.** Audible credentials live ONLY in
  `shim`. The Rust app talks to the shim over loopback; it never
  parses RSA private keys or Amazon cookies itself. If the shim is
  compromised or restarted, the app keeps working with cached library
  data; only new fetches fail.
- **Two NAS trees, never mixed.**
  - `SCRIBE_LIBRARY_DIR` holds canonical `Author/Title/Title.m4b` —
    audiobookshelf points at this and only this.
  - `SCRIBE_ORIGINAL_DIR` holds the untouched `.aaxc`/`.aax` files as
    downloaded from Audible plus their `.voucher` sidecars. ABS never
    sees this tree. Useful for re-converting without re-downloading.
- **Pi memory budget.** 1 GB total, shared with halo/chat/abs/kanidm/etc.
  Scribe must not buffer book files in RAM. All large IO is streamed —
  CDN → press → NAS — via `tokio::io::copy` over reqwest byte streams.
- **Polling.** New-book poll interval defaults to 5 minutes
  (`SCRIBE_POLL_INTERVAL_MIN`). Uses `?sort_by=-PurchaseDate&num_results=10`
  cheap call. Smart-backoff overnight (00:00–06:00 local → 30 min) added if
  it ever rate-limits.
- **Design system.** Tokens copied verbatim from halo's `themes.ts`. Don't
  fork — when halo evolves, copy across so the family stays aligned. Only
  intentional divergence is `Wordmark.tsx` (quill stroke + dot).

## Working on this repo

- App at `:3003`, shim at `127.0.0.1:3004` (Pi). Press at `:3005` (mini, behind Caddy).
- Frontend dev server at `:5173`; Vite proxies `/api`, `/auth`, `/status`
  through to `:3003`.
- TanStack Router `routeTree.gen.ts` is regenerated by the Vite plugin
  on every dev/build. Committed copy is a stub so a fresh clone type-checks
  before the plugin runs.
- Use `DEV_AUTH=1` locally — kanidm not required.

## Out of scope (for now)

- Reading from accounts other than Audible (Libro.fm, Storytel, etc.)
- Re-encoding M4B (lossless remux only)
- Playback inside scribe (audiobookshelf + listen-this handle that)
- Multi-user libraries — each user keeps their own Audible accounts
- Mobile / native app (web UI only, designed touch-friendly)

If a feature crosses into those areas, raise before implementing.
